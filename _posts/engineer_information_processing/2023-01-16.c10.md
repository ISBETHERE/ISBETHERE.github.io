## 네트워크 침해 공격 관련 용어

```

ping of Death
(죽음의 핑)

ping 명령을 전송할때 패킷의 크기를
인터넷 프로토콜 허용범위이상으로 공격 대상의
네트워크를 마비시키는 서비스 거부 공격 방법

SMURFING
스머핑
IP나 ICM의 특성을 악용하여 엄청난 양의 데이터를 한 사이트에
집중적으로 보냄으롴서 네트워크 불능 상태로 만드는 공격

스미싱
문자 메시지(SMS)를 이용해 사용자의 개인 신용정보를 
빼내는 기법

피싱
개인 정보를 낚시 합성어로
이메일이나 메시전 등을 공기관이나 금융기관을 사칭하여
개인정보를 빼내는 기법

ping Flood
ICMP 메세지를 보내 이에 대한 응답으로
시스템 자원을 모두 사용하게 해 시스템이 정상적으로 동작하지 못하도록
공격하는 방법


EVil Twin 
실제 존재하는 동일한 이름의 무선 WIFI 신호를 송출하여
로그온한 사람들의 계정 정보나 신용 정보등을 뺴내는 기법

스위치 재밍
위조된 매체 접근 제어(MAC) 주소를 지속적으로
네트워크로 흘려보내 ,스위치 MAC 주소 테이블의 저장기능을
혼란시켜 더미허브처럼 작동하게 하는 공격방법


```

##입력 데이터 보안 약점

```
SQL 삽입
SQL을 삽입하여 내부 데이터베이스 서버의 데이터를 유출 및 변조하고
관리자 인증을 우회하는 보안 약점

경로 조작 및 자원 삽입
데이터 입출력 경로를 조작하여 서버 자원을 수정,삭제할수있는 보안약점

크로스사이트스크립팅
웹페이지에 악의적인 스크립트를 삽입하여 방문자들의 정보를 탈취하거나
비정상적인 기능 수행을 유발하는 보안 약점

운영체제 명령어 삽입
외부 입력값을 통해 시스템 명령어의 실행을 유도함으로써
권한을 탈취하거나 시스템 장애를 유발하는 보안약점

위험한 형식 파일 업로드
외부 입력값을 통해 시스템 명령어의실행을 유도함으로써
권한을 탈취하거나 시스템 장애를 유발하는 보안 약점

신뢰되지 않는 URL 주소로 자동접속 연결
입력값으로 사이트 주소를 받는 경우 이를 조작하여
방문자를 피싱 사이트로 유도하는 보안약점

메모리 버터 오버플로
연속된 메모리 공간을 사용하는 프로그램에서 할당된 메모리의 범위를
넘어선 위치에서 자료를 읽거나 쓰려고 할때 발생하는 보안 약점


```

## 칩입 탐지 시스템

```
컴퓨터 시스템의 비정상적인 사용, 오용, 남용 등을 실시간으로 탐지하는 시스템

-방화벽 차단으로 100% 해결 불가능

- 내 외부 실시간 추적과 유해 정보감시가 필요

-오용탐지
미리 입력해 둔 공격 패턴이 감지되면 이를 알려줌

-이상탐지
평균적인 시스템의 상태를 기준으로 비정상적인 행위나 자원의 사용이 감지되면
이를 알려줌

-침입 탐지 시스템의 종류

HIDS( Host-Based Intrusion Detection)

시스템의 내부를 감시하고 분석하는데 중점을 둔 침입 탐지 시스템이다.


NIDS( Network-Based Intrusion Detection)

외부로 부터 침입을 감시하고 분석하는데 중점을 둔 탐지시스템.


-침입 탐지시스템위치 5가지

패킷이 라우터로 들어오기전
 
라우터 뒤

방화벽 뒤

내부 네트워크

DMZ 


```
